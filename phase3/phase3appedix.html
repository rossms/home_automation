Evaluation Plan

The evaluation plan looks to meet all the requirements defined in early phases. Subjects representing each persona will be selected as evaluators. An evaluation form will be provided to collect information (negative or positive) for all subjects designated to evaluate the prototype. Information collected will be used as direct feedback on improvements or areas needing to be corrected. The following requirements define the success criteria for all subjects performing evaluations.

  • Product design must integrate multiple common household activities reducing the need for them to be performed individually.
  • Product design must incorporate economic and social factors introduced by the user groups.
  • Product design must simplify a task by reducing the number of steps and effort needed to accomplish it.
  • Product design results in energy/cost savings for the end user.

Assessment Feedback Results

Evaluator 1: Single Family Home (Working Mother) Age: Mid-30’s
Errors: During this evaluation, no errors were experienced.
Attention: I found a lot of detail in the application which made it visually understandable.
Fatigue: No fatigue experienced. The tasks were very direct and not prolonged.
Productivity: The application would have enabled my productivity just by allowing management of many devices from centralized mobile interface.
Efficiency: The application seemed to provide results quickly and was very responsive. Overall, this will be allowing more efficiency in my day as a single parent.
Ease of Use: The application and interface was very easy to use
Culture: The application had a very similar interface to many mobile applications. Was not different or anything new to learn.
Emotion: I felt this is a new way for doing things. Much easier and pleasurable way for doing things – meaning was no discomfort in controls or way for managing home appliances. Great to know or gain insights into billing and usage of electricity.
Experience: Great experience. Site was very responsive. Can see where this can be expanded upon to other devices and use cases.
Convenience: By bringing the interfaces onto a mobile platform, I can see the state or access devices from anywhere. Do have concerns around security but overall was very easy to use.
Togetherness: The flow was very put together. Limited steps to access or navigate between windows.
Empathy: How much will something like this be? Is it affordable for non-tech savvy families?
Sociability: I can see this expanding into a social platform. Conservative, technology enthusiast’s groups
Entertainment: None at this point was connected.
Gamification: Great innovation. Can track trends for improvement. Would add functionality for some form of scoring around savings to make more game like.

Errors: Some confusion as to the back button on various pages. "Where is it supposed to go?" Was ultimately able to navigate with little difficulty.
Attention: The fan control page did not indicate which fan was where.
Fatigue: Controls were sensible and easy to use with the exception of the back button comments already recorded. User did not experience fatigue.
Productivity: "This would probably save me time and money."
Efficiency: See fatigue and Productivity comments
